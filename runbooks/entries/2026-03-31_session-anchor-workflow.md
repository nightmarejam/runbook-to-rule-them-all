## Intent

Run one high-value work session without getting trapped in either rigid planning or reactive context switching.
This workflow is designed for pattern-driven and ADHD activation styles: start from pull, constrain scope,
produce one proof artifact, and park everything else.

## Prerequisites

- You have a clear area of pull (example: "unblock pushes", "prepare CPA handoff", "stabilize monitoring").
- You can capture notes quickly (RunBook entry, scratch file, or terminal notes).
- You can stop after one artifact is complete (not after every possible improvement).

## Steps

1. Write a one-sentence anchor for this session.
   - Format: "By end of session, I will `<outcome>`."
   - Example: "By end of session, I will restore GitHub push capability for `ai-stack`."
2. Define one proof artifact before doing work.
   - Examples: successful `git push` output, CSV exports generated, dashboard URL with live data.
3. Set a three-move limit.
   - Write exactly three moves you are allowed to do before reassessing.
   - If a move explodes in scope, split it and keep only the highest-leverage sub-step.
4. Execute move 1 immediately (no extra planning).
5. Keep a parking lot as new ideas appear.
   - Add every side quest to parking lot instead of switching.
   - Only switch if it blocks the current anchor directly.
6. Reassess after move 3.
   - If proof artifact is done: stop and record closure.
   - If not done: create one additional three-move block.
7. Close with a short session log.
   - What was done, what failed, what is parked, what is next.

## Verification

- Anchor sentence is completed or explicitly reframed.
- Proof artifact exists and is inspectable by another person.
- Parking lot captured at least one deferred item (if any emerged).
- Session ends with a next-anchor suggestion, not an open cognitive loop.

## Known failure modes

- Over-scoping: trying to solve architecture, implementation, and cleanup in one pass.
- Tool drift: running diagnostics endlessly without committing to an artifact.
- Perfection trap: polishing docs/UI when the anchor was operational.
- Silent context switching: starting a "quick fix" that becomes the new session.
- False done: saying complete without a concrete proof artifact.

## Notes

This method intentionally supports non-linear cognition:
- Emergence is allowed in exploration, constrained in execution.
- Curiosity is not suppressed; it is staged via parking lot.
- Discipline is applied to closure, not to ideation.
