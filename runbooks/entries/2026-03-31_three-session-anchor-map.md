## Intent

Run meaningful work without calendar rigidity by using session anchors, proof artifacts,
and strict stop rules. This map converts broad project pressure into three executable sessions.

## Prerequisites

- Access to all three repos:
  - `/home/jonat/ai-stack`
  - `/home/jonat/ai-stack/projects/constella-framework`
  - `/home/jonat/ai-stack/runbook-to-rule-them-all`
- Git SSH auth working for pushes
- A place to capture parking-lot items

## Steps

1. Session 1: Unblock trust
   - Anchor: restore remote safety and push reliability
   - Proof artifact: successful push outputs + clean status snapshot
   - Moves:
     1) Resolve push-protection unblock flow for `ai-stack`
     2) Push pending runbook/constella commits
     3) Commit `.gitignore` hardening as separate change
2. Session 2: CPA/FreshBooks handoff readiness
   - Anchor: make financial handoff immediately usable
   - Proof artifact: sent-ready package and validated import files
   - Moves:
     1) Finalize verified COGS and K-1 split numbers
     2) Validate FreshBooks CSVs and mapping assumptions
     3) Prepare/send Lisa update with exact figures and attachments
3. Session 3: Direction lock (Constella x RunBook)
   - Anchor: define responsibilities and next build lane
   - Proof artifact: committed direction doc + next two runbooks
   - Moves:
     1) Write ownership boundary (`Constella` vs `RunBook`)
     2) Define ChromaDB/knowledge contract at policy level
     3) Add two high-reuse runbook entries from current pain points
4. Use the stop rule each session:
   - Stop when proof artifact exists
   - Do not extend the session just because more ideas appeared
5. Capture all side pulls in parking lot, not execution stream.

## Verification

- Each session has one explicit anchor sentence
- Each session ends with one concrete proof artifact
- Parking lot is updated and carried forward
- At least one commit or externally verifiable output per session

## Run history

## Known failure modes

- Starting Session 2 or 3 before Session 1 trust loop is restored
- Conflating ideation with execution and missing artifacts
- Expanding one session into a multi-day unresolved thread
- Allowing dashboards/docs polish to replace operational closure

This map is built for non-linear cognition:

- Discovery can stay emergent

- Closure is measured by artifact, not mood or time spent
